Contributing to the Handbook
Basic information on contributing to the GitLab Handbook.
Note
Please see the
repository contributing file
for information on the
Developer Certificate of Origin + License
when contributing to GitLab.
You can help to improve the
GitLab Handbook
website by submitting merge requests to
the handbook project
.
Thank you for your interest and help!
We don’t mind people inside and outside the company making suggestions to change significant things.
Feel free to make a proposal, we can discuss anything. If we don’t agree, we’ll feel free not to merge it, and we’ll thank you for caring about it.
We want to create a welcoming environment for everyone who is interested in contributing. Please visit our
Code of Conduct page
to learn more about our commitment to an open and welcoming environment.
